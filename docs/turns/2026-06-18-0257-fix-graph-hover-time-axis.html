Fix Graph Hover, Time Direction, and Axis Scale

Adjusted the dashboard graph so hover details appear only while hovering, time reads left to right with the newest day on the right, and the y-axis shows a visible metric basis.

2026-06-18 02:57 dirtydash-2n0 dashboard chart ux

Summary

The ledger still lists daily rows newest first for fast scanning, but the graph now renders those same days chronologically so the current day lands on the right. The tooltip no longer shows by default on the selected day. The selected-day sessions panel also has a little more top breathing room.

Changes Made

  • Created a chart-only chronological copy of the daily rows with [...summary.daily].reverse().
  • Mapped the selected day against the chart row order so the active bar or point still follows selection correctly.
  • Changed the chart tooltip so it renders only when hovered has a concrete row index.
  • Added y-axis tick labels derived from the active metric, with compact token values in token and cache modes.
  • Changed the x-axis labels to show the first and last chart days around the active metric name.
  • Added targeted spacing under .sessions-pane .pane-title.
  • Rebuilt dashboard/dist so the embedded static app matches the source.

Context

The previous chart used the same newest-first row order as the ledger. That made the latest day sit on the left, which fought the usual mental model of time moving forward left to right. The previous tooltip also fell back to the selected day when nothing was hovered, so it looked permanently active.

Important Implementation Details

Ledger order is preserved The table remains newest first, because that order is useful for scanning recent usage.
Graph order is visual Only the chart receives reversed rows, so API shape and table behavior do not change.
Tooltip state is explicit null means no hover, so the info panel stays hidden until pointer hover.
Axis labels follow sort mode cost uses dollars; tokens and cache use compact token counts.

Expected Impact for End-Users

  • The latest usage day now appears at the right edge of the graph, matching the direction users expect time to move.
  • The chart no longer looks like it is constantly showing hover details when the pointer is elsewhere.
  • Token graphs now have visible y-axis values, making bar and point height easier to interpret.
  • The selected-day sessions table has more room below its title, so the pane feels less cramped.

Issues, Limitations, and Mitigations

  • No screenshot-based visual check was completed because the local Playwright browser binary is missing. The build and source checks passed.
  • Keyboard focus no longer opens the mouseover tooltip. The chart marks still keep their aria-label values for assistive context, and pointer click selection remains unchanged.
  • The generated dist filenames changed because Vite hashes bundled assets. That is expected when the embedded dashboard is rebuilt.
