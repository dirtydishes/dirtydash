DirtyDash now imports the nested pi-agent/Codex usage shape correctly, trusts source-reported costs when present, and repairs stale zero-cost rows on reimport instead of leaving them stuck behind the old INSERT OR IGNORE behavior.

Summary

The fix addresses the known undercount where pi-agent and Codex-derived rows could carry millions of tokens but stay at $0.00. DirtyDash now extracts the direct nested usage counters, uses reported totals from usage.cost.total, aliases Codex-flavored OpenAI providers to bundled OpenAI pricing, and updates existing imported rows when their pricing fields have changed.

Changes Made

  • Replaced import-only insertion with upsert_usage_event, returning inserted, updated, or skipped so stale pricing can be repaired.
  • Added recursive direct usage parsing for pi-agent shapes such as input, output, cacheRead, cacheWrite, and reasoning.
  • Prefer source-reported cost totals from usage.cost.total and tag those events with reported-cost.
  • Added OpenAI provider/model alias lookup so openai-codex and gpt-5.4-spark can resolve against bundled OpenAI pricing.
  • Updated CLI import reporting to include existing rows that were updated.
  • Updated the dashboard copy and Burn Report so it surfaces unpriced tokens rather than a hardcoded unknown-spend summary.
  • Rebuilt the bundled dashboard assets under dashboard/dist.

Context

The live DirtyDash database matched the screenshot before the fix: about $316.95 total, with pi-agent contributing roughly 176.1M tokens but $0.00 cost. The root problem was twofold: the parser was finding token values in a way that could miss the real direct usage object, and old imported rows were ignored on reimport because the hash already existed.

A clean temporary import after the fix priced those pi-agent rows as reported-cost and raised the local DirtyDash total to $456.81. That confirms the zero-cost-row bug is fixed, but it does not yet explain CodexBar's approximately $1204 30-day total. That remaining reconciliation is tracked as dirtydash-q9g.

Important Implementation Details

Reported Cost Wins

When an event includes a finite nonnegative usage.cost.total, DirtyDash stores that value directly, sets confidence to 0.98, and records reported-cost as the pricing version. If no reported total exists, it falls back to bundled/manual pricing.

Existing Rows Are Repairable

The import path now compares the pricing-relevant state for an existing raw_event_hash. If tokens, model, cost, confidence, or pricing version differ, DirtyDash updates the row in place and reports it as updated.

Alias Pricing Is Conservative

Provider aliases only map known Codex/OpenAI forms to openai. Model candidates keep the original name first, then try version/spark-stripped variants such as gpt-5.4 for gpt-5.4-spark.
