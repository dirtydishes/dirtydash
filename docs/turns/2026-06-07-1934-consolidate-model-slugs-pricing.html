Consolidate Model Slugs And Pricing

Model totals now read as model names instead of provider-prefixed harness labels, while fast-mode slugs stay separate and bundled GPT pricing follows the Cursor model table semantics more closely.

Summary

Updated Dirtydash so model breakdown rows show canonical model slugs such as gpt-5.5 and gpt-5.5-fast, without the user-facing openai/ or openai-codex/ prefix. OpenAI/Codex provider aliases still merge internally when they represent the same model.

Changes Made

  • Replaced provider/model string grouping for model totals with a dedicated model aggregation path.
  • Kept fast-mode models as distinct rows, so gpt-5.5-fast does not roll into gpt-5.5.
  • Added Cursor doc-slug aliasing, so pricing lookup can handle forms like gpt-5-5-fast and gpt-5.5-fast.
  • Updated bundled OpenAI pricing to the 2026-06-07 Cursor-derived snapshot, including explicit gpt-5.5-fast and gpt-5.5-long rows.
  • Changed bundled pricing seeding so refreshed bundled rows update existing non-override records while preserving manual overrides and local/free markings.

Context

The Models page had rows like openai/gpt-5.5, openai-codex/gpt-5.5, and openai/gpt-5.4-fast. That made harness naming more prominent than the model itself. The user clarified that the provider prefix is not useful on this surface and that fast mode should remain visible as its own model row.

Important Implementation Details

  • The database still stores provider and model exactly as imported. The new rollup happens in dashboard_summary(), so existing imports benefit immediately.
  • The display label is the model slug only. Rows from openai and openai-codex with the same model merge because their final label matches.
  • Fast mode remains distinct because its slug remains distinct: -fast is not stripped during summary grouping.
  • GPT cache-write pricing is now zero in the bundled Cursor snapshot rows. Cache reads still use the discounted Cursor/OpenAI cache-read rate.

Expected Impact for End-Users

The Models page should now feel much less noisy: rows are model names, not harness/provider labels. Users can still distinguish fast-mode usage because -fast remains its own row, and cost estimates should better match Cursor’s published model pricing behavior.

Validation

  • Passed: cargo test
  • Passed: npm run build in dashboard/
  • Passed: manual browser check at http://127.0.0.1:4601; Models rows showed gpt-5.5, gpt-5.4, gpt-5.4-fast, and no provider prefixes.

Issues, Limitations, and Mitigations

  • Pricing is still a bundled snapshot, not live-fetching Cursor at runtime. The snapshot version and source label make that explicit.
  • Provider is intentionally hidden in model totals. If two providers use the exact same model slug in the future, this view will merge them; detailed session rows still keep provider information.
  • Long-context detection is not automatic yet. A gpt-5.5-long pricing row exists, but imported usage must identify that model/slug to use it.
