dirtydash V1 local-first foundation

Implemented the first working scaffold for the V1 plan: Rust CLI, SQLite storage, local importers, bundled pricing, SSH remote configuration, embedded React/Vite dashboard, tests, and visual verification.

Summary

Built a narrow but real V1 foundation for dirtydash. The CLI can scan, import, serve, run doctor checks, manage pricing overrides, and configure pull-style remotes. Usage events land in SQLite with provenance and idempotent raw event hashes.

Changes Made

  • Added a Rust workspace with the dirtydash binary and core modules for CLI, config, app paths, SQLite, importers, pricing, remotes, and server.
  • Implemented source scanning and metadata-only import for Claude Code, Codex, OpenCode, and pi-agent paths.
  • Added SQLite migrations for usage events, source files, pricing records, remotes, and schema migrations.
  • Added bundled pricing records with manual override and mark-free commands.
  • Built a dark, dense React/Vite dashboard and embedded the built assets in the Rust server.
  • Added unit and CLI tests for importers, malformed records, idempotent import, pricing, scan, import, doctor, pricing list, and serve startup.

Context

The V1 plan values correctness, provenance, and a usable local dashboard over broad remote operations. The implementation follows that posture: local import and local dashboard are real, while remote sync is intentionally conservative and does read-only SSH file discovery without installing anything remotely.

Importer path assumptions were checked against ccusage source documentation for Claude Code, Codex, OpenCode, and pi-agent.

Important Implementation Details

IdempotencyEach event is keyed by raw_event_hash so repeated imports skip existing rows.
Codex deltasCodex token_count entries are treated as cumulative and stored as per-event deltas.
PricingBundled records are seeded once; manual overrides replace records in SQLite.
PrivacyThe default path stores metadata and provenance, not raw conversation previews.

Expected Impact for End-Users

Users can run dirtydash scan, dirtydash import, and dirtydash serve to get a local dashboard that answers basic spend, token, cache, source, model, project, session, and provenance questions from SQLite-backed metadata.

Issues, Limitations, and Mitigations

  • Remote sync currently configures remotes and performs read-only SSH file discovery, but it does not yet import remote files. Follow-up: dirtydash-fol.
  • Importer fixtures are synthetic and flexible by design. Real-world fixture hardening is tracked in dirtydash-20p.
  • Unknown models are imported with cost $0 and lower confidence rather than guessed pricing. Users can correct this with dirtydash pricing override or dirtydash pricing mark-free.

Follow-up Work
