Ledger At-A-Glance Summary

Added a compact cumulative usage strip above Ledger, backed it with reasoning-effort aggregation, and tightened the dashboard's sort, session naming, and keyboard hints.

2026-06-18 11:42 dirtydash-ute dashboard usage summary

Summary

The Ledger page now leads with the numbers users check first: cumulative cost, cumulative tokens, the most-used model share, reasoning-effort share, and cache read share. The summary is compact, visible before the trace/detail work area, and uses real imported usage data.

Changes Made

  • Added reasoning_effort to imported usage events, SQLite migration, upsert comparison, and dashboard summary aggregation.
  • Carried Codex reasoning effort from turn_context rows onto later token-count usage rows.
  • Added a Ledger usage-glance strip for cumulative cost, tokens, top model percentage, top reasoning-effort percentage, and cache read share.
  • Changed sort modes to day, cost, and tokens, with day descending as the default.
  • Made session rows use human-readable date/time labels while preserving raw session IDs in hover and provenance details.
  • Added hover shortcut hints and made Tab cycle workspace views outside Ledger.
  • Removed numbered page labels from the top page title.
  • Rebuilt dashboard/dist for the embedded Rust dashboard.

Context

Dirtydash is a local-first usage instrument. The Ledger page needs to answer "what did this cost and what used the tokens?" immediately, before the user drills into rows, traces, pricing, or provenance. The previous layout had the data available, but the most important cumulative numbers were not promoted as a first-read surface.

Important Implementation Details

Reasoning effort is stored Imported events now persist normalized effort values: minimal, low, medium, or high.
Codex context carries forward Codex token rows inherit effort from the latest turn_context, matching the existing model and turn-id parser behavior.
Percentages are token shares The top model and top reasoning effort percentages are calculated against cumulative imported tokens.
Tab is view navigation On Sources, Sessions, and Ops, Tab and Shift+Tab cycle the active view instead of walking every focusable control.

Expected Impact for End-Users

Users can now land on Ledger and immediately see cumulative spend and token volume, plus the dominant model and reasoning-effort mix. Navigation should also feel calmer because page titles are cleaner, session names scan as timestamps, and tabbing moves between views instead of bouncing through every row and button.

Validation

  • Ran npm run build in dashboard/.
  • Ran cargo test; all 22 Rust tests passed.
  • Served the real embedded dashboard with cargo run -- serve --port 0.
  • Used Playwright with system Chrome to capture desktop and mobile screenshots.
  • Verified Tab cycles Ops from import to pricing and Shift+Tab returns to import.
  • Verified the mobile viewport reported no document-level horizontal overflow.

Issues, Limitations, and Mitigations

  • Existing imported rows will show unknown reasoning effort until the user reimports sources that contain effort metadata.
  • The top model and reasoning percentages are token-share percentages, not event-count percentages. This is intentional because the Ledger is usage-centered.
  • The bundled dashboard assets changed filenames as part of the production build. The document omits those generated hunks to keep review focused.

Follow-up Work

  • No new follow-up issue was filed. A useful later improvement would be a small import hint when reasoning effort is mostly unknown.
