Dirtydash TUI Dashboard Redesign

Replaced the production dashboard with a dense, keyboard-first TUI surface centered on the daily usage ledger and real selected-day session drilldowns.

2026-06-18 02:37 dirtydash-883 dashboard backend api

Summary

The app now opens to 1 Ledger, with daily usage sorted newest first, a custom SVG chart, selected-day sessions, and an inspector showing provenance and pricing assumptions. The old 13-page sidebar and /mock1 through /mock4 mock-route mode were removed from the React app.

Changes Made

  • Added backend support for date-descending daily ledger rows and /api/days/:day/sessions.
  • Rebuilt the dashboard around four numbered workspaces: Ledger, Sources, Sessions, and Ops.
  • Added secondary tabs, command/status bar, search, keyboard shortcuts, selected day and session cursors, sort mode, view mode, and chart type state.
  • Added custom SVG bar, line, and histogram chart modes with hover/focus tooltips and selected-point tracking.
  • Reworked CSS around panes, grid lines, sticky dense rows, terminal-like selection, dark OKLCH tokens, responsive scroll containment, and reduced-motion fallbacks.
  • Rebuilt dashboard/dist so the Rust binary embeds the redesigned UI.

Context

Dirtydash is a local-first usage instrument for developers who already trust terminal workflows. The redesign keeps trust markers close to the data: parser, pricing snapshot, raw path, cache share, fast/priority usage, stale imports, parse errors, and doctor warnings remain visible where they affect interpretation.

Important Implementation Details

Newest-day ledger Daily rollups now use a dedicated SQL query ordered by date descending instead of the generic spend-sorted grouping helper.
Real drilldown The selected day fetches real grouped sessions from /api/days/:day/sessions, limited to 100 rows.
Keyboard guard Shortcuts skip inputs and editable targets, so typing in search does not trigger workspace or sort changes.
Responsive density Dense rows keep min-widths inside scrollable panes while the page shell clips horizontal overflow on mobile.

Expected Impact for End-Users

Users land on the most recent usage day first, can move through the ledger with j/k, switch workspaces with 1 through 4, and inspect real sessions for the selected day without leaving the main dashboard. Trust details stay visible without long explanatory copy.

Validation

  • npm run build in dashboard/ passed.
  • cargo test passed: 19 unit tests, 2 CLI tests, and doc tests.
  • Browser smoke against cargo run -- serve --host 127.0.0.1 --port 5174 passed with no console errors.
  • Manual browser automation confirmed default screen is 1 Ledger, SVG chart is present, the ledger has newest day first, and selected-day sessions render from the real API.
  • Keyboard smoke confirmed g, j, 1-4, /, ?, and Esc; shortcuts do not fire while typing in search.
  • Mobile smoke at 390px confirmed no page-level horizontal overflow, with dense rows scrollable inside panes.
  • Reduced-motion smoke confirmed animation duration collapsed to 1e-05s.

Issues, Limitations, and Mitigations

  • The session cache sort uses a confidence-weighted proxy because SessionSummary does not currently expose grouped cache tokens. The daily ledger and chart use real cache read tokens.
  • The matrix remains an aggregate source/model scan rather than a fully crossed source-model backend query. It uses existing summary data to avoid broad API restructuring.
  • Raw paths can be long. The panes keep them readable with wrapping and internal overflow rather than expanding the whole page.

Follow-up Work

  • Add session-level cache token fields to the backend if cache sorting inside the Sessions workspace needs to be exact.
  • Add a real source-model cross-tab API if the matrix should stop using proportional aggregate hints.
  • Consider a small focused visual regression snapshot for the ledger shell once the app has a stable fixture database.
