Pane-Focused Keyboard Navigation

Changed Ledger keyboard behavior so selection starts in daily usage, Tab cycles the active pane, and keys operate inside the active pane without moving the page.

2026-06-18 03:52 dirtydash-715 dashboard keyboard ux

Summary

The Ledger workspace now has an explicit active pane model. It starts on daily usage, uses Tab and Shift+Tab to cycle through chart, sessions, and inspector, and keeps all scrolling inside the selected pane.

Changes Made

  • Added LedgerPaneId and activePane state, defaulting to daily.
  • Made Ledger panes focusable with data-pane, tabIndex, and active-pane styling.
  • Changed Tab handling on Ledger to cycle panes instead of moving browser focus through row buttons.
  • Scoped arrow behavior to the active pane: daily and chart move selected day, sessions moves selected session, inspector scrolls.
  • Added PageUp and PageDown to scroll the active Ledger pane.
  • Updated shortcut copy to document pane navigation and pane scrolling.
  • Rebuilt dashboard/dist for the embedded Rust dashboard.

Context

The fixed viewport is now the right shell behavior, but the app also needs a clear keyboard focus model. Previously, Ledger arrow keys always preferred the selected-day sessions collection. That made the first selection feel wrong because the user enters Ledger looking at daily usage first.

Important Implementation Details

Daily starts active Ledger initializes and resets to the daily pane when entering the workspace.
Tab cycles panes Tab advances through daily, chart, sessions, and inspector. Shift+Tab reverses that order.
Rows stay visible The selected daily row and selected session row are scrolled into view inside their own panes.
Page scroll stays locked Pane-local scrolling uses the active pane element, so window.scrollY remains zero.

Expected Impact for End-Users

Ledger keyboard control now starts where the user is looking: daily usage. Users can press Tab to move pane focus, then use arrows or page keys inside that pane without losing the fixed-screen dashboard.

Issues, Limitations, and Mitigations

  • This pass scopes pane cycling to Ledger, where the multi-pane keyboard model is currently most important.
  • Chart pane arrow keys currently move day selection, matching the chart's selected-day role. It does not add chart-specific point focus beyond the existing SVG point affordances.
  • The browser verification used Chrome DevTools directly because this repository does not have Playwright installed.
